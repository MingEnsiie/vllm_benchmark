# `../Assets/models/` 与 `../Assets/datasets/` 实验选型参考手册

更新时间：`2026-04-10`

适用范围：
- 本文只覆盖当前仓库上一级 `../Assets/models/` 与 `../Assets/datasets/` 目录。
- 数据集部分同时纳入已经确定要接入的新 benchmark 规划；未实际下载的条目会明确标注为“待接入”。
- 客观字段优先来自本地 `README.md`、`config.json`、`preprocessor_config.json`、权重文件清单与数据文件结构。
- “推荐结论”是结合本地资产形态、官方模型卡、当前仓库单卡部署上下文给出的手册结论，不等同于统一官方结论。

## 1. 先看结论：按实验场景怎么选

### 1.1 通用图文主评测

推荐优先级：
1. `Qwen3.5-35B-A3B-GPTQ-Int4`
2. `gemma-4-31B-it`
3. `gemma-4-26B-A4B-it`
4. `Qwen3.5-27b-GPTQ-Int4`

推荐数据集：
- `MMMU-Pro`
- `LongBench v2`
- `IFEval`

为什么这样选：
- `MMMU-Pro` 负责多模态主 benchmark。
- `LongBench v2` 和 `IFEval` 分别补长文理解与指令跟随，避免主评测只剩图文问答。
- `Qwen3.5-35B-A3B-GPTQ-Int4` 是当前仓库里“参数规模大 + MoE 激活参数低 + 已量化”的主力模型，适合做高性价比主评测。
- `gemma-4-31B-it` 和 `gemma-4-26B-A4B-it` 都是 256K 上下文的图文模型，其中 `26B-A4B` 更偏吞吐/成本均衡，`31B` 更偏上限。
- `Qwen3.5-27b-GPTQ-Int4` 不是 MoE，但量化后本地体积可控，适合作为 Qwen 系 dense 对照组。

不建议：
- 直接拿 `Qwen3.5-0.8B`、`Qwen3.5-2B` 做最终主结论模型，它们更适合 smoke test 或流程验证。

### 1.2 文档理解 / OCR

推荐优先级：
1. `gemma-4-31B-it`
2. `gemma-4-26B-A4B-it`
3. `Qwen3-Omni-30B-A3B-Instruct`
4. `Qwen3.5-35B-A3B-GPTQ-Int4`

推荐数据集：
- `OmniDocBench`
- `MMMU-Pro`

为什么这样选：
- `OmniDocBench` 是新的文档理解 / OCR 核心 benchmark，覆盖更完整的文档解析场景。
- `MMMU-Pro` 可作为文档能力之外的通用图文补充对照。
- Gemma 4 官方模型卡明确强调文档/PDF 解析、OCR、手写识别、图像理解；对文档类实验更直接。
- `Qwen3-Omni-30B-A3B-Instruct` 支持文本、图像、音频、视频输入以及文本/语音输出，如果后续实验会扩展到语音讲解、音视频问答，它的可扩展性最好。
- `Qwen3.5-35B-A3B-GPTQ-Int4` 适合作为单卡可跑的文档/OCR 强基线。

不建议：
- 把 `Gemma-4-31B-JANG_4M-CRACK` 当作正式文档/OCR 评测基线。它是研究性质的“去拒绝 + 混合比特”变体，目标与标准能力评测不一致。

### 1.3 图表理解

推荐优先级：
1. `gemma-4-31B-it`
2. `Qwen3.5-35B-A3B-GPTQ-Int4`
3. `gemma-4-26B-A4B-it`
4. `Qwen3.5-9B`

推荐数据集：
- `OmniDocBench`
- `MMMU-Pro`

为什么这样选：
- 当前新的 benchmark 基线里没有单独保留 `ChartQA`，图表能力更建议先纳入 `OmniDocBench` 与通用图文主集统一管理。
- 如果后续需要做更细的图表专项，再回补 `ChartQA` 作为历史专项资产。
- Gemma 4 官方卡显式提到 chart comprehension；Qwen3.5 官方卡强调 unified vision-language foundation，适合做第二主线。
- `Qwen3.5-9B` 是更省资源的中等规模选择，适合在正式大模型跑前快速筛 prompt 与评估流程。

### 1.4 数学视觉推理

推荐优先级：
1. `gemma-4-31B-it`
2. `Qwen3.5-35B-A3B-GPTQ-Int4`
3. `gemma-4-26B-A4B-it`
4. `Qwen3.5-27b-GPTQ-Int4`

推荐数据集：
- `MMMU-Pro`
- `LongBench v2`

为什么这样选：
- `MMMU-Pro` 仍然是数学视觉和多学科视觉推理的主入口。
- `LongBench v2` 可以补足长内容总结、跨段推理与长上下文问答，对复杂推理实验更有代表性。
- Gemma 4 官方卡公开给出 `MATH-Vision`、`MMMU Pro` 等视觉推理指标，31B 与 26B-A4B 都适合做正式对比。

### 1.5 轻量快速验证 / 流程调试

推荐优先级：
1. `Qwen3.5-4B`
2. `Qwen3.5-9B`
3. `gemma-4-E4B-it`
4. `Qwen3.5-2B`

推荐数据集：
- `IFEval`
- `MRCR v2`
- `MMMU-Pro` 的小样本切片

为什么这样选：
- 这几个组合都能较快暴露 prompt、解析、打分和服务稳定性问题。
- `IFEval` 和 `MRCR v2` 更容易快速暴露指令跟随、精确检索和长上下文处理问题。
- `MMMU-Pro` 可抽少量样本做图文链路 smoke test。
- `gemma-4-E4B-it` 和 `Qwen3.5-4B` 都属于中小型多模态模型，适合做成本敏感实验。

### 1.6 音频 / 视频 / 全模态扩展实验

推荐优先级：
1. `Qwen3-Omni-30B-A3B-Instruct`
2. `gemma-4-E4B-it`
3. `gemma-4-E2B-it`

推荐数据集：
- `Video-MME`
- 可先拿 `MMMU-Pro` / `IFEval` 复用图文与文本链路做回归，再补完整视频流程

为什么这样选：
- `Qwen3-Omni-30B-A3B-Instruct` 是当前仓库里唯一明确支持文本、图像、音频、视频输入，并支持文本与自然语音输出的模型。
- `gemma-4-E2B-it`、`gemma-4-E4B-it` 支持文本、图像、音频输入与文本输出，更适合轻量音频实验。

### 1.7 长上下文与单卡部署

推荐优先级：
1. `Qwen3.5-35B-A3B-GPTQ-Int4`
2. `Qwen3.5-27b-GPTQ-Int4`
3. `gemma-4-26B-A4B-it`
4. `gemma-4-31B-it`

为什么这样选：
- Qwen3.5 主线模型普遍是 `262,144` 原生上下文，量化版更适合本地部署。
- Gemma 4 的 `26B-A4B` 与 `31B` 也是 `256K`，但本地权重体积更大。
- 当前仓库已有基准报告显示：`Qwen3.5-35B-A3B-GPTQ-Int4` 在单卡上做 `128K` 级长上下文时，吞吐基本被 prefill 成本主导，实际应优先采用低并发，尤其是 `1` 并发。

### 2.1 模型对比矩阵

| `Qwen3.5-9B` | Qwen3.5 | Dense 9B | 文本、图像 | 文本 | 262K | 否 | 17.98 GiB | 成本与效果平衡、图文基线 | 部署成本高于 4B，性能仍低于大模型 |

| `Gemma-4-31B-JANG_4M-CRACK` | Gemma 4 研究变体 | Dense 31B，JANG 混合比特 | 文本、图像 | 文本 | 256K | 混合比特，README 标称均值 5.1 bit | 21.10 GiB | 安全研究、非常规对照 | 明确“去拒绝/CRACK”目标，不适合标准能力基线 |

### 2.2 模型观察与选型建议

- `Qwen3.5` 系列是当前仓库最完整的图文主线，`0.8B/2B/4B/9B/27B/35B-A3B` 形成了从轻量到正式主力的完整梯度。
- `Qwen3.5-35B-A3B-GPTQ-Int4` 的优势在于：MoE 激活参数低、已量化、本地体积比部分 dense 大模型更友好，适合单卡上做正式主评测。
- `Gemma 4` 系列的优势在于：官方模型卡对图像理解、OCR、文档解析、图表和数学视觉的能力描述更具体，适合作为文档/OCR/数学视觉强基线。
- `Qwen3-Omni-30B-A3B-Instruct` 不应作为第一优先主榜模型，但它是未来扩展到音频、视频、语音输出时最重要的储备模型。
- `Qwopus3.5-27B-v3` 适合作为“官方模型之外的研究变体对照组”，尤其是编码和推理导向实验；不建议把它当成主榜唯一结论模型。
- `Gemma-4-31B-JANG_4M-CRACK` 应单列为“研究/安全向特殊模型”，不要与标准指令模型混作主结论。

### 2.3 本地部署提示

- 当前仓库唯一已有性能报告的是 `Qwen3.5-35B-A3B-GPTQ-Int4`。
- 该模型在短上下文下，`8` 并发吞吐最高；在 `128K` 长上下文下，并发几乎不提升总吞吐，`1` 并发更合理。
- 因此，大模型长上下文实验应优先控制并发，而不是盲目拉高 QPS。

### 3.2 各数据集说明

#### `MMMU-Pro`

- 当前文档将其视为新的多模态主 benchmark。
- 需要和本地已有 `../Assets/datasets/mmmu` 明确区分：旧目录记录的是 `MMMU` 资产，不应直接替代 `MMMU-Pro` 结果。
- 接入前需确认：
  - 官方仓库中的 `Pro` 配置、子任务或版本映射
  - 是否复用现有 `eval_mmmu.py`，还是单独维护脚本
- 适合做：
  - 多学科正式主评测
  - 通用图文模型主结论
  - 多模态主 benchmark

#### `OmniDocBench`

- 官方仓库说明其覆盖文档解析真实场景，包含多种文档类型、布局和语言类型。
- 适合统一承接原先分散在 `DocVQA`、`OCRBench`、`ChartQA` 一类文档 / OCR / 版式能力需求。
- 接入前需优先确认：
  - license 与下载方式
  - 是按页面、元素、问答还是解析任务来打分
- 适合做：
  - 文档理解 / OCR 核心评测
  - 表格、公式、版式、段落等文档结构能力对比

#### `LongBench v2`

- 主定位不是视觉，而是长文档理解、长内容问答与总结。
- 适合补足多模态主 benchmark 之外的长上下文能力判断。
- 适合做：
  - 长内容问答
  - 长文总结
  - 跨段信息整合

#### `IFEval`

- 主要衡量模型是否遵守自然语言指令和输出约束。
- 虽然不是多模态 benchmark，但对于线上稳定输出非常关键。
- 适合做：
  - 指令跟随
  - 格式约束测试
  - 稳定性回归

#### `Video-MME`

- 当前推荐只在有视频场景时启用。
- 需要先核对官方 license、视频来源限制和本地存储成本。
- 适合做：
  - 视频理解
  - 时序推理
  - 全模态链路回归

#### `BFCL-V4`

- 核心不是静态问答，而是函数调用 / 工具调用 / agentic tool use。
- 接入前需确认评测运行时是否依赖联网、外部搜索 API 或执行环境。
- 适合做：
  - Agent 基础能力
  - Tool-use schema 对齐
  - 调用格式稳定性

#### `Tau2 (avg over 3)`

- 更偏系统级 agent 任务完成度。
- 文档里按“avg over 3” 记录，意味着结果通常需要多次运行取平均。
- 适合做：
  - 多步任务完成
  - agent 端到端能力
  - 系统级评测补充

#### `MRCR v2`

- 强调长上下文中的精准找点，而不是泛化式总结。
- 当前先作为长上下文检索专项保留，版本细节后续再核对。
- 适合做：
  - 精准检索
  - 长上下文定位
  - retrieval stress test

#### `MMMLU`

- 主要用于多语言理解与国际化能力评测。
- 没有国际化需求时，可排在后续选测。
- 适合做：
  - 多语言对比
  - 国际化基线
  - 语言迁移能力观察

## 4. 推荐实验配方

### 4.1 快速冒烟

- 模型：`Qwen3.5-4B` 或 `gemma-4-E4B-it`
- 数据集：`IFEval` + `MRCR v2` + `MMMU-Pro` 小样本
- 目标：先验证文本链路、长上下文链路、图文链路与基本评测脚本

### 4.2 通用图文主评测

- 模型：`Qwen3.5-35B-A3B-GPTQ-Int4`、`gemma-4-31B-it`、`gemma-4-26B-A4B-it`
- 数据集：`MMMU-Pro` + `LongBench v2` + `IFEval`
- 目标：形成主结论，并同时覆盖多模态主 benchmark、长内容能力和输出稳定性

### 4.3 文档 / OCR 专项

- 模型：`gemma-4-31B-it`、`gemma-4-26B-A4B-it`、`Qwen3-Omni-30B-A3B-Instruct`
- 数据集：`OmniDocBench`
- 目标：统一看文档理解、OCR、表格、公式和版式等核心文档能力

### 4.4 工具调用与系统级专项

- 模型：`gemma-4-31B-it`、`Qwen3.5-35B-A3B-GPTQ-Int4`
- 数据集：`BFCL-V4` + `Tau2 (avg over 3)`
- 目标：区分 tool-use 基础能力与系统级 agent 完成度

### 4.5 视频与成本敏感对比

- 模型：`Qwen3.5-4B`、`Qwen3.5-9B`、`gemma-4-E4B-it`
- 数据集：`Video-MME` 或 `MMMU-Pro` 小样本 + `IFEval`
- 目标：有视频场景时验证端到端成本；无视频场景时快速找到单位成本最优解

## 5. 最终建议

- 如果只能先挑一个主力模型：优先 `Qwen3.5-35B-A3B-GPTQ-Int4`。
